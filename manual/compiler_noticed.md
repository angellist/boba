## Noticed

`Tapioca::Dsl::Compilers::Noticed` decorates RBI files for subclasses
of `Noticed::Event` and `Noticed::Ephemeral`.

For example, with the following notifier class:

~~~rb
class NewCommentNotifier < Noticed::Event
  required_params :comment
  deliver_by :email
end
~~~

This compiler will produce the RBI file `new_comment_notifier.rbi` with the following content:

~~~rbi
# new_comment_notifier.rbi
# typed: true
class NewCommentNotifier
  class << self
    sig { params(params: T::Hash[Symbol, T.untyped]).returns(T.class_of(NewCommentNotifier)) }
    def with(params); end

    sig { params(recipients: T.untyped, enqueue_job: T.nilable(T::Boolean), options: T.untyped).void }
    def deliver(recipients = T.unsafe(nil), enqueue_job: T.unsafe(nil), **options); end

    sig { params(recipients: T.untyped, enqueue_job: T.nilable(T::Boolean), options: T.untyped).void }
    def deliver_later(recipients = T.unsafe(nil), enqueue_job: T.unsafe(nil), **options); end
  end
end
~~~

### Why This Compiler Exists

The base Noticed gem RBI (generated by `bin/tapioca gem noticed`) provides `with`, `deliver`, and `deliver_later` methods on `Noticed::Deliverable::ClassMethods`, but with generic return types. This compiler provides:

1. **Subclass-specific return types**: The `with` method returns `T.class_of(SpecificNotifier)` enabling proper method chaining with type safety
2. **ActiveRecord CTE conflict resolution**: Tapioca's ActiveRecord compiler generates a `with` method for CTEs that shadows Noticed's `with` method; this compiler ensures the correct signature is available
